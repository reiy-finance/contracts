## Fixes

### Fixed — Zombie object after full partial-drain (Medium)

`intent_book::consume_intent_partial` now requires `fill_amount < remaining` (error
`EFillNotStrictlyPartial`); a fill that would consume the whole balance must go through
`take_intent_full`, which deletes the object. `fill_status` is therefore always `PartialFill` on
this path — no zero-balance object can survive.

Regression: `test_full_drain_via_partial_rejected` (full-drain aborts),
`test_strict_partial_fill_keeps_residual` (strict partial keeps the residual).

### Fixed — Expired winning intent unfairly slashed (Medium)

`IntentMeta` gained a `deadline` field (threaded through `record_intent_meta`, `requeue_intent`,
`intent_meta_of`). `trigger_fallback` now slashes and re-queues only intents that were still
settleable at the deadline (`now <= deadline`). Intents that expired before settlement are neither
slashed nor re-queued — the owner reclaims them via `cancel_intent` in a later epoch. Solvers are
punished only for genuine timeouts they could have avoided.

Regression: `test_expired_intent_not_slashed_in_fallback` (bond intact, no slash recorded).
Unchanged: `test_expired_winning_intent_take_aborts` (take still rejects an expired intent).

### Fixed — Overpaid `close_batch` fee not refunded (Low)

`close_batch` now splits exactly `required_fee` off the supplied coin, deposits that, and refunds
the remainder to the caller (`#[allow(lint(self_transfer))]`).

Regression: `test_overpaid_fee_refunded` (treasury collects only the required amount).
